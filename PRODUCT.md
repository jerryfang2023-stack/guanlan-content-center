# Product

## Register

product

## Users

内容工厂的内容服务对象是对 AI 有兴趣、但充满焦虑的企业老板、创业者和业务负责人。他们迫切想知道企业 AI 化该从哪里开始、钱该花在哪里、哪些流程先改、如何避免买错工具。

直接使用者是观澜 AI 自有内容团队。观澜 AI 在内容中扮演老板企业 AI 化路上的顾问和伙伴，负责把 AI-Radar 每日材料转成老板能理解、能行动、能复盘的选题、公众号草稿和发布包。

## Product Purpose

内容工厂 Lite 按内容生命周期组织为四个一级栏目：今日工作台、内容资产、编辑发布和发布中心。今日工作台读取 AI-Radar 的商业信号、一线观点和社群情报，由“澜”完成选题判断、写作与修订；人工保留选题、事实、发布和学习四个关键确认点。内容资产统一呈现选题、草稿、待发布和已发布记录；选题库与写作风格作为次级管理视图。编辑发布在同一栏目完成 Markdown、主题、HTML、校验、预览和交付；写作与排版仍通过不可变快照隔离，防止排版回写原草稿。发布中心统一管理待发布和发布复盘。经典写作台在过渡期保留为编辑发布的次级入口，不再占用一级导航。

今日工作台通过本地 Node 服务调用已登录的 Claude CLI，由“澜”读取内容中心当前日期下的真实每日选题，并执行用户在对话中提出的要求，不预置写作计划、任务队列或自动排期。浏览器不得直接执行 CLI；服务端不传入 system prompt，不覆盖或限制 Claude CLI 的默认系统、CLAUDE.md、工具、权限模式、模型、费用上限、插件、Skills、Hooks、MCP、子 Agent、命令或用户/项目配置，也不对正式任务设置执行超时、输出大小或单任务并发锁。页面只通过 JSON Schema 约定最终返回字段，以便把结果写回内容中心。用户可选择已发布写作风格，也可让“澜”从参考文或写作要求中提炼并保存新的可编辑风格方法与 Prompt。生成结果保存到内容工厂本地工作区，并直接交给编辑发布；需要时仍可在经典写作台继续修改。

## Brand Personality

克制、判断型、专业可信。界面应有观澜 AI 的品牌识别，但不能像营销页；它是一个每天会被反复使用的生产工具。

## Anti-references

不要做成静态展示页、营销风格落地页或装饰过重的卡片墙。不要在选题阶段提前混入平台、风格、篇幅等写作配置，也不要在选题视图改写 AI-Radar 原始标题；标题改写属于与澜写作或经典写作台。不要把公众号、小红书、抖音塞进一个通用写作表单。

## Core Page Principle

**非必要不展示。** 每个页面只保留完成当前任务所需的信息、状态和操作。能由结构、默认值或上下文表达的内容不重复展示；辅助文案必须解决真实疑问，否则删除。新增或修改页面时，先证明字段、说明和操作的必要性，再决定是否展示。

## Design Principles

- 非必要不展示：优先删除重复字段、解释性副文案、可由上下文推断的信息和低频干扰项。
- 老板视角优先：先判断这条材料能否回答老板的焦虑、痛点、经营需求和下一步动作。
- 先判断，再生成：选题页只做内容决策与分流。
- 状态要可执行：每个标签都应该说明下一步去向，而不是抽象评价。
- 对话工作区要稳定：长对话只在消息区内部滚动，输入区始终可达，不得让消息持续撑长整个页面。
- 每日更新，长期沉淀：当日选题按日期组织，潜力选题可以进入长期选题库。
- 候选池要足够宽：每天至少提供 10 条以上原始选题供人工判断。
- 排期与事实分离：月度内容日历使用 `scheduledDate`，观澜事实保留 `sourceDate`、信号 ID 与原始链接；不得把历史信号标成排期日当天的新观察。
- 平台流程独立：公众号、小红书、抖音各自承载不同流程，不共享一个聚合写作界面。
- 风格与稿件分离：风格是可版本化的方法和 Prompt，不是一篇范文；参考文案仅用于提炼，草稿修改不得静默覆盖已发布风格。
- 外部 Skill 只内化方法：保留直接来源、Stars 快照和许可提醒，不直接复制第三方 Prompt 或特征句。
- 写作与排版解耦：写作页的唯一正文真源是 Markdown；每次交接生成不可变版本快照，排版页只消费快照，不回写或覆盖原草稿。
- 图片交付可选择：无配图需求时允许明确跳过并进入排版；需要配图时，服务端只负责生成结构化视觉方案，浏览器将方案渲染为真实 PNG，也支持导入替换。图片资产保留内容指令、视觉方案、版本、状态和采用关系，二进制统一存入 IndexedDB。正文或标题改变后，旧图必须重新生成或明确采用才能进入排版交接。
- 静态排版图片闭环：已托管的 HTTPS 配图可按 Markdown 插图位自动进入 HTML；本地资产通过“插入正文位置 → 复制图片 → 公众号编辑器粘贴”完成安全交付。
- 发布复盘可回填：发布平台、日期、阅读、收藏、评论、线索、复盘结论和下一步动作均保存在本地工作区。
- 标题边界清晰：选题视图只引用原始标题，与澜写作或经典写作台可以生成和编辑文章标题草稿。
- 三条管线分层使用：商业信号做事实底座，一线观点做趋势和语境线索，社群情报做痛点和需求线索。
- 保持观澜气质：用克制的色彩、明确的信息层级和稳定的工具型交互表达专业判断。

## Accessibility & Inclusion

以 PC 端工具界面为主，保持清晰对比度、可键盘聚焦状态、明确按钮标签，并避免依赖颜色作为唯一状态提示。
